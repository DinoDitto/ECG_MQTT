## 0. 開場（30 秒）

大家好，我這次的專案是一套 **基於 IoT 的即時心電圖監測系統**。

作業需求是：以 **AD8232 ECG 感測器** 為 Sensor，把心電圖資料傳到 **MQTT Server**，
再透過 **Node-RED** 顯示心電圖波形，並在訊號超出設定值時啟動警示流程。

我除了完成這些需求，還額外做了一個 **網頁健康數據管理平台**，可以儲存歷史心律並做多人管理。

---

## 1. 系統架構（45 秒）

[可秀架構圖或 README]

整個系統的資料流是這樣：

1. **AD8232 感測器** 從身體擷取心電訊號
2. 送到 **ESP32**，由它做 **數位濾波、R 波偵測、計算心率（BPM）**
3. ESP32 透過 WiFi，用 **MQTT 協定** 把資料發佈到電腦上的 **Mosquitto Broker**
4. Broker 再把資料分送給兩個訂閱端：
   - **Node-RED**：顯示即時波形、BPM 儀表、警示
   - **網頁平台**：把數據存進 SQLite 資料庫，做歷史查詢與管理

這裡用 MQTT 的好處是**發佈/訂閱架構**：ESP32 只負責發資料，
不需要知道誰在看，所以波形、網頁、資料庫可以同時接收，彼此獨立。

## 2. 硬體與韌體（45 秒）

[展示麵包板實體]

硬體核心是 **ESP32**，搭配：
- **AD8232 ECG Kit** 擷取心電訊號
- **1602 LCD** 顯示即時狀態與心率
- **被動式蜂鳴器**，每偵測到一次心跳就嗶一聲，警報時連續嗶
- **一顆按鈕** 控制開始 / 停止測量

韌體的演算法重點：
- 用 **IIR 濾波器** 去除基線漂移
- **動態門檻 + 三點波峰偵測** 找出 R 波（每一次心跳）
- 再用 **3 拍平均** 算出穩定的 BPM
- 當 BPM 超過 **130**，就發出警報

## 4. 技術亮點與結語（30 秒）

總結一下這個專案的特色：

1. **完整的端到端 IoT 鏈路**：感測器 → 邊緣運算 → MQTT → 視覺化 + 資料庫
2. **MQTT 一對多架構**：同一份資料同時供 Node-RED 與網頁使用
3. **超出作業需求**：除了 Node-RED 警示，還做了資料持久化與多人管理後台
4. 全部程式碼都放在 GitHub 上版控管理

需要說明的是，這是**單導程、學術用途**的系統，不能用於臨床診斷。

## 附：可能被問到的問題

**Q：為什麼 BPM 有時會跳很高？**
A：ECG 訊號容易受肌肉動作與電源雜訊干擾，體動時會產生假的尖峰。
   實務上靠濾波與不應期降低誤判，量測時保持放鬆不動可得到穩定數值。

**Q：為什麼用 MQTT 不直接用 HTTP？**
A：MQTT 輕量、適合 IoT 裝置，且發佈/訂閱架構讓多個接收端能同時取得資料，
   ESP32 不需為每個接收端各開一條連線。

**Q：警示流程在哪裡判斷？**
A：兩層。ESP32 端先判斷是否超過閾值並發出 ecg/alert；
   Node-RED 收到後，由 Function 節點再判斷並觸發畫面警報通知。
