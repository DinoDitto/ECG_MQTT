01 專案簡介

本系統是一套基於 IoT 的即時心電圖（ECG）監測平台。使用 AD8232 心電感測器 擷取心臟電訊號， 由 ESP32 進行數位濾波與心律計算後，透過 MQTT 協定 傳送至電腦， 再以 Node-RED 顯示即時波形並執行警示流程，同時由 網頁平台 儲存與管理健康數據。

作業需求對應：以 AD8232 為 Sensor → 傳到 MQTT Server → Node-RED 顯示心電圖 → 訊號超出設定值（130 BPM）即啟動警示流程。本專案全數達成，並額外實作資料庫與管理後台。

採用 MQTT 的 發佈/訂閱（Pub/Sub）架構：ESP32 只負責發佈資料，無需知道接收端是誰， 因此 Node-RED 與網頁平台可同時訂閱同一份資料，彼此完全獨立。

03 功能特色

📈

即時心電波形

Node-RED Dashboard 以 100Hz 顯示連續 ECG 波形，可見每次心跳的 R 波尖峰。

💓

心律計算

IIR 濾波 + 動態門檻波峰偵測 + 3 拍平均，計算穩定 BPM。

🚨

警示流程

BPM 超過 130 即觸發：蜂鳴器連嗶、LCD 顯示 ALERT、Node-RED 彈出警報。

🖥️

本地顯示

1602 LCD 即時顯示狀態與心率，無需連網也能看。

🗄️

資料持久化

心律、警報寫入 SQLite，可查歷史趨勢與統計。

👥

多人管理後台

管理員可檢視所有使用者數據，架構預留多裝置擴展。

05 訊號處理演算法

步驟方法參數
① DC 移除IIR 低通濾波α = 0.995
② 雜訊追蹤動態背景強度估計α = 0.95
③ 觸發門檻nodc_level + 偏移+ 8.0
④ 波峰偵測三點局部最大值n1>n2 & n1>n0 & >門檻
⑤ 不應期避免重複觸發250 ms
⑥ RR 驗證合理間距範圍270~2000 ms
⑦ N 拍平均BPM = 3×60000 / ΣRR3 拍
⑧ 警報超過閾值發送BPM > 130

06 啟動步驟

  1. 啟動 MQTT BrokerMosquitto（D:\mosquitto），1883 埠，已允許區網連線
  2. 啟動 Web 平台cd ECG_Web → npm start → http://localhost:3000
  3. 啟動 Node-REDnode-red → 匯入 flows.json → Deploy → http://localhost:1880/ui
  4. 燒錄 ESP32PlatformIO → esp32dev → Upload
  5. 開始測量貼好電極 → 按藍色按鈕 → 在 Dashboard 看即時波形與 BPM
